User Timing

Test Description Section Contact
test_user_timing_clear_marks This test validates that the performance.clearMarks() method is working properly. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_clear_measures This test validates that the performance.clearMeasures() method is working properly. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_exists This test validates that all of the methods used to interact with the User Timing API are defined. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_mark This test validates that the performance.mark() method is working properly. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_mark_exceptions This test validates that the performance.mark() method throws a SYNTAX_ERR exception whenever a navigation timing attribute is provided for the name parameter. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_measure This test validates that the performance.measure() method is working properly. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_measure_exceptions This test validates that the performance.measure() method throws a SYNTAX_ERR exception whenever a non-existent mark is provided as the startMark or endMark, and the method also throws a INVALID_ACCESS_ERR whenever a navigation timing attribute with a value of zero is provided as the startMark or endMark. 4.2 Extensions to the Performance Interface Web Performance Working Group
test_user_timing_measure_navigation_timing This test validates that the performance.measure() method is working properly when navigation timing attributes are used in place of mark names. 4.2 Extensions to the Performance Interface Web Performance Working Group
idlharness This test validates the WebIDL included in the user Timing specification. 4.2 Extensions to the Performance Interface Web Performance Working Group

Page Visibility

Test Description Section Contact
test_attributes_exist This test validates that all of the attributes associated with the Page Visibility feature exist (but does not validate that their values are correct). 4.2 Extensions to the Document interface Web Performance Working Group
test_default_view This test validates that document.hidden == false and document.visibilityState == "visible" for windowless subdocuments. 4.2 Extensions to the Document interface Web Performance Working Group
test_child_document This test validates that, within child documents, all of the Page Visibility API attributes exist, are read-only, and match the value of the attributes within the parent document. 4.2 Extensions to the Document interface Web Performance Working Group
test_minimize This test validates that the page properly becomes hidden and visible due to minimizing, maximizing, and restoring the browser window. 4.2 Extensions to the Document interface Web Performance Working Group
test_read_only This test validates that the Page Visibility attributes are read only. 4.2 Extensions to the Document interface Web Performance Working Group
test_tab_state_change This test validates that the page properly becomes hidden and visible due to switching tabs. 4.2 Extensions to the Document interface Web Performance Working Group
idlharness This test validates the WebIDL included in the Page Visibility specification. 4.2 Extensions to the Document interface Web Performance Working Group

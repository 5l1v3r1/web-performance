Web Performance Working Group

Our Mission

We provide methods to measure and improve aspects of application performance of user agent features and APIs.

The Co-Chairs of the Working Group are Ilya Grigorik and Todd Reifsteck. The W3C Team Contacts for the Web Performance Working Group are Philippe Le Hégaret and Xiaoqian Wu.

We have a primer for Web Performance Timing APIs. That's a good introduction for you.

Real-world RAIL (Response, Animation, Idle, Load)

Ilya Grigorik presented on Quantify and improve real-world RAIL during the Chrome Dev Summit 2015. Check it out as a good introduction to the work done in the Web Performance Working Group.

High Resolution Time Level 2 is ready

We intend to ship a revision of High Resolution Time that includes a proper definition for time origin and worker support. Check it out and send us feedback.

What is the Performance Timeline?

Methods to store and retrieve high resolution performance metric data in a common performance timeline. See also our primer.
